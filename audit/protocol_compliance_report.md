## Known Protocol Deviations

### ISSUE-255: Non-Fatal Tool Errors (Documented)
- **Behavior**: When `MCP_NON_FATAL_TOOL_ERRORS=true` (default), some error responses may not set `isError` flag
- **Rationale**: Keeps conversation flow recoverable for expected states (auth required, quota exceeded)
- **Control**: `MCP_NON_FATAL_TOOL_ERRORS=false` enforces strict MCP behavior
- **Documentation**: `src/schemas/handler-deviations.ts:162-180`
- **Assessment**: Low risk. Opt-in deviation with clear override.

### collaborate Schema Workaround (SDK Bug)
- **Behavior**: `sheets_collaborate` uses flat `z.object()` + `.superRefine()` instead of `z.discriminatedUnion()`
- **Cause**: MCP SDK v1.26.0 bug with large discriminated unions (41 actions)
- **Risk**: Schema validates correctly but uses non-standard pattern
- **Test coverage**: `tests/contracts/collaborate-discriminated-union.test.ts` validates behavior
- **Fix when**: SDK bug resolved in future version
